# Validation Playbooks
**최종 갱신:** 2026-04

## 검증 원칙
기회를 찾는 것보다 빠르게 틀리는 것이 더 가치 있다. 이 문서는 점수가 높은 상위 기회에 공통으로 적용 가능한 90일 검증 사이클과 도메인별 변형을 정의한다.

## 90일 검증 플레이북 (Top 5 기회 공통 템플릿)

### Phase 1 (Day 1-30): 구매자 탐색
목표: 돈 낼 사람이 실제로 존재하는지 확인.

| 활동 | 방법 | 성공 기준 |
|---|---|---|
| 구매자 인터뷰 10명 | LinkedIn, 네트워크, 콜드 이메일 | 10명 중 7명 이상 "이건 실제 문제다" |
| 현재 해결 방식 조사 | 인터뷰 + 공개 자료 | 기존 솔루션의 명확한 한계 식별 |
| 예산 주체 확인 | 조직도·구매 프로세스 파악 | 승인권자 1명 이상 특정 |
| 요금 의향 탐색 | 직접 질문 또는 가격 제시 반응 | 1명 이상 구체적 금액 제시 |

### Phase 2 (Day 31-60): MVP 제작 및 테스트
목표: 가장 작은 형태로 가치 전달 가능한지 확인.

| 활동 | 방법 | 성공 기준 |
|---|---|---|
| MVP 제작 | 보고서/대시보드/워크숍 중 1가지 | 8주 이내 완성 |
| 파일럿 고객 2-3명 | 무료 또는 할인 제공 | 2명 이상 사용 및 피드백 제공 |
| 반복 주기 | 피드백 수렴 → 수정 | 최소 1회 개선 사이클 완료 |

### Phase 3 (Day 61-90): 유료화 전환
목표: 지불 의사를 실제 결제로 전환.

| 활동 | 방법 | 성공 기준 |
|---|---|---|
| 유료 전환 시도 | 가격 제시 → 계약서 or 인보이스 | 1건 이상 유료 전환 |
| 리텐션 신호 확인 | 재구매·추천 의향 조사 | 50% 이상 재구매 의향 |
| 유닛 이코노믹스 | CAC vs LTV 추정 | LTV/CAC > 3x 가능성 확인 |

## 킬 기준 (공통)
다음 중 하나라도 해당하면 즉시 중단한다:
- 구매자 인터뷰 10명 중 8명 이상이 "긴급하지 않다"고 응답
- 예산 주체가 불명확하거나 의사결정 사이클이 18개월 초과
- 기존 대기업 솔루션이 80% 이상 문제를 해결 중
- MVP 제작 비용이 첫 계약 예상 금액의 50% 초과
- 데이터 접근이 법적으로 불가능

## 도메인별 검증 특이사항

| 도메인 | 핵심 검증 질문 | 최적 파일럿 고객 유형 |
|---|---|---|
| AI 인프라/전력 | "지금 부지 선정 지연이 몇 개월인가?" | 데이터센터 개발사 전략팀 |
| 반도체/공급망 | "대만 충돌 시나리오를 이사회에 보고했는가?" | 대기업 구매/리스크팀 |
| 기후/보험 | "기후 리스크가 PF 승인에 영향주는가?" | 보험사 언더라이터, PF팀 |
| 지정학/방산 | "공급망 지정학 리스크 담당자가 있는가?" | 방산/방위산업 전략팀 |
| Sovereign AI | "AI 규제 대응 예산이 배정됐는가?" | 규제대응팀, 법무팀 |
| 한국 특화 | "한국 정부 과제로 연결 가능한가?" | 연구기관, 대기업 미래전략실 |

## 운영 규율
- 90일 사이클은 단일 기회에 한 번만 적용한다. 동일 기회에 두 번째 사이클을 돌리려면 가설을 반드시 변경한다.
- Phase 전환 시점에서 성공 기준 미달이면 다음 Phase로 진입하지 않고 킬 기준을 재검토한다.
- 모든 인터뷰 노트와 가격 제시 반응은 별도 raw 파일로 저장하고 본 문서에는 요약만 남긴다.
