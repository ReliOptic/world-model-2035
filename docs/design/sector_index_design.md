# Pages Design: Sector Index and Domain Entry Points

**정보 신선도:** 🟢  |  **최종 갱신:** 2026-05  |  **다음 갱신:** 2026-08  
**Issue scope:** #31, #38. Design only; implementation follows the navigation PRD.

## Sector index grouping
Initial grouping:

- Foundations and methodology
- Climate and nature
- Technology
- Energy
- Space
- Geopolitics
- Defense
- Economics and labor
- Corporate roadmaps
- International organizations
- Science
- Scenarios and predictions

## Card/list fields
Each sector entry should show:

- **sector** name
- **core question** for 2035
- **latest freshness** status
- **key bottleneck** or constraint
- **prediction-log link** when applicable
- primary **roadmap** or overview file
- source/audit status when available

## Domain entry point behavior
Clicking a sector should lead to the sector's roadmap or overview, plus a “How to use this sector” block based on the onboarding template.

## Data source direction
A future implementation can maintain a small manifest or derive entries from markdown metadata. The first implementation should prefer an explicit manifest for auditability.
