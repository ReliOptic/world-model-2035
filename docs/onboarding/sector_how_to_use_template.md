# Sector Template: How to use this sector

**정보 신선도:** 🟢  |  **최종 갱신:** 2026-05  |  **다음 갱신:** 2026-08  
**Issue scope:** #31, #34. Copy this block into sector-level entry docs when a domain needs reader guidance.

## How to use this sector
Use this sector to answer one question:

> What constrains the 2035 normal path for this domain, and how far is current expectation from that path?

## Current state
- What is true now?
- Which source establishes it?
- How fresh is the source?

## Official roadmap
- What do governments, firms, standards bodies, or institutions claim will happen?
- Which milestone is official rather than repo inference?
- Which milestone should be discounted for execution risk?

## Base / Upside / Downside
- **Base:** normal constrained path using current evidence.
- **Upside:** faster adoption, bottleneck relief, or policy acceleration.
- **Downside:** slower adoption, binding bottlenecks, regulatory delay, or capital shortage.

## Bottlenecks
Identify the three constraints most likely to bend the sector path:

1. physical or infrastructure constraint
2. policy/regulatory constraint
3. capital, labor, supply-chain, or social-acceptance constraint

## Prediction candidates
Every material forecast update should ask:

- Is there a measurable claim?
- What is the deadline?
- What source will score it?
- Should it be added to the prediction log?

## When to open a sector issue
Open an issue when:

- a source is stale or materially superseded
- a sector lacks Base / Upside / Downside logic
- a bottleneck is missing or under-specified
- a new forecast should affect the prediction log
- a sector becomes a dependency for FLP or scenario work

## Minimum close evidence
- changed files
- source list
- audit commands
- prediction-log impact
- known gaps
