# 자원 좌표축 고정 + 슬라이스는 강조만 가변 (Coupling Map 비교가능성)

Map Slice는 시나리오마다 재도출되지만, **자원 어휘목록을 안정 좌표축으로 고정**하고 슬라이스는 *그 고정 축 위에서* 이번 시나리오가 어디를 binding으로 만드는지 **강조(가중)만 바꾼다**. 좌표계를 매번 새로 만들지 않는다. 이유: 슬라이스가 좌표를 자유롭게 바꾸면 두 시나리오를 교차 비교할 수 없고, 시나리오가 쌓여도 누적 통찰이 안 생긴다(살아있는 엔진의 핵심 가치 상실). FLP가 "고정 수식 버전 + 고정 정규화 앵커 → 같은 입력 같은 출력"으로 푼 규율을 지도에도 적용한다.

**자원 어휘 v1 (좌표축, 버전관리):** 전력 · compute(GPU/HBM) · 자본·조달 · 핵심광물(구리) · 숙련 인지노동. 새 자원(물·변압기·데이터 등)이 시나리오에서 나오면 축에 *추가*하고 버전업(v2…), 과거 버전은 FLP처럼 보존.

## Considered Options
- **자유 슬라이스:** 시나리오마다 schema에서 자유롭게 5×5 선택. 거부 — 교차 비교 불가, 누적 통찰 불가.

## Consequences
- "5×5가 시나리오마다 바뀐다" = *강조*가 바뀐다는 뜻이지 좌표계가 매번 새로 생기는 게 아니다.
- 메타 통찰 가능: "시나리오 N개 중 M개에서 전력이 Hub로 떴다" 같은 누적 패턴.
